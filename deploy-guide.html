Prerequisitos

Cuentas y herramientas requeridas antes de comenzar
  • 🐙
    Repositorio en GitHub El código debe estar en un repo de GitHub (o GitLab/Bitbucket). Vercel lo conecta directamente. Si aún no está: git remote add origin https://github.com/tu-usuario/wc26-pool.git
  • 🟢
    Cuenta Supabase Crea una cuenta en supabase.com. El plan Free incluye 2 proyectos activos, 500 MB de DB y 1 GB de almacenamiento — suficiente para el torneo.
  • ▲
    Cuenta Vercel Crea una cuenta en vercel.com y conéctala a tu cuenta de GitHub. El plan Hobby es gratuito e incluye cron jobs en proyectos con el add-on de Cron.
  • ⚽
    API-Football (opcional) Para sync automático de resultados en vivo. Crea cuenta en api-football.com → plan Basic (~$10/mes) o usa el Free tier (100 req/día, suficiente para pruebas).
2

Supabase en producción

Crear el proyecto de producción y obtener las credenciales
  • 1️⃣
    Crear proyecto En supabase.com/dashboard → New project. Elige una región cercana a tu audiencia (us-east-1 para México/USA/Canadá). Guarda bien la Database Password.
  • 2️⃣
    Obtener credenciales En el proyecto creado: Settings → API. Copia los tres valores que necesitarás: Project URL, anon public key y service_role key.
  • 3️⃣
    Configurar Auth En Authentication → URL Configuration: agrega tu dominio de Vercel (https://tu-app.vercel.app) en Site URL y en Redirect URLs. Puedes editar esto después de obtener la URL de Vercel.
  • 4️⃣
    Habilitar Email Auth En Authentication → Providers → Email: asegúrate que esté habilitado. Para producción desactiva "Confirm email" o configura un dominio SMTP personalizado para los correos de confirmación.
3

Ejecutar migraciones

Aplicar schema, equipos y partidos en la base de datos de producción

Las tres migraciones en supabase/migrations/ deben ejecutarse en orden. Hay dos formas:

Opción A — Supabase CLI (recomendado)

$ npm install -g supabase

# Enlaza con tu proyecto de producción
$ supabase link --project-ref <tu-project-ref>

# Ejecuta todas las migraciones
$ supabase db push

El project-ref está en la URL de tu dashboard: supabase.com/dashboard/project/abc123xyz

Opción B — SQL Editor manual

En tu dashboard de Supabase, ve a SQL Editor y ejecuta los archivos en este orden:

  • 📄
    001_schema.sql Crea todas las tablas, tipos, funciones, triggers y políticas RLS. Ejecutar primero — el resto depende de él.
  • 🌎
    002_seed_teams.sql Inserta los 48 equipos clasificados al Mundial 2026 con sus grupos (A–L) y códigos FIFA.
  • 📅
    003_seed_matches.sql Inserta los 72 partidos de la fase de grupos con fechas y sedes. Si tienes la llave eliminatoria confirmada, agrégala aquí.
⚠️

No ejecutes en orden incorrecto. El 003_seed_matches.sql hace referencia a los equipos del 002. Si hay error de FK, verifica que el 002 se ejecutó correctamente primero.

4

Variables de entorno

Todas las variables que necesita la aplicación en producción

Estas son las variables que debes configurar en Vercel (Project → Settings → Environment Variables). Márcalas para los entornos Production, Preview y Development.

Variable Dónde obtenerla Req.
NEXT_PUBLIC_SUPABASE_URL Supabase → Settings → API → Project URL Sí
NEXT_PUBLIC_SUPABASE_ANON_KEY Supabase → Settings → API → anon public Sí
SUPABASE_SERVICE_ROLE_KEY Supabase → Settings → API → service_role (secreto) Sí
CRON_SECRET Genera tú mismo: mínimo 32 chars aleatorios Sí
API_FOOTBALL_KEY api-football.com → Dashboard → My Account → API Key Opt.

Generar CRON_SECRET

# En tu terminal local:
$ openssl rand -hex 32

# O en Node.js:
$ node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
🔒

SUPABASE_SERVICE_ROLE_KEY tiene acceso total a la base de datos, ignorando RLS. Nunca la expongas al cliente ni la comitas al repositorio. Úsala solo en variables de entorno del servidor.

5

Desplegar en Vercel

Conectar el repositorio y hacer el primer deploy
  • 1️⃣
    Importar proyecto En vercel.com/new → Import Git Repository → selecciona el repositorio de GitHub. Si no aparece, instala la integración de GitHub en Vercel.
  • 2️⃣
    Configurar el proyecto Framework: Next.js (detectado automáticamente). Build command: npm run build. Output directory: déjalo en blanco (Next.js lo maneja).
  • 3️⃣
    Agregar variables de entorno Antes de hacer click en Deploy, expande Environment Variables y agrega todas las variables del paso 4. También puedes agregarlas después desde Settings.
  • 4️⃣
    Primer deploy Haz click en Deploy. Vercel construirá y desplegará automáticamente. El proceso toma 1–3 minutos. Obtendrás una URL *.vercel.app.
  • 5️⃣
    Actualizar Supabase con la URL Copia tu URL de Vercel y agrégala en Supabase → Authentication → URL Configuration → Site URL y también en Redirect URLs.
ℹ️

Deploys automáticos: cada push a main dispara un nuevo deploy en producción. Los PRs crean entornos de Preview con su propia URL temporal.

Re-deploy tras cambiar variables

Si agregas o cambias variables de entorno después del primer deploy, necesitas hacer un re-deploy manual para que tomen efecto:

$ vercel --prod

# O desde el dashboard: Deployments → ⋯ → Redeploy

Cron jobs

Sincronización automática de resultados cada 5 minutos

El proyecto ya incluye vercel.json con la configuración del cron job:

// vercel.json
{
  "crons": [
    {
      "path": "/api/cron/sync",
      "schedule": "*/5 * * * *"
    }
  ]
}

Vercel ejecuta /api/cron/sync cada 5 minutos durante el torneo. El endpoint verifica el header Authorization: Bearer <CRON_SECRET> para rechazar llamadas no autorizadas.

  • ⚙️
    Verificar en Vercel En tu proyecto → Cron Jobs (en la barra lateral). Verás el cron listado con la última ejecución y su status. Si no aparece, asegúrate de que el vercel.json esté en la raíz del proyecto.
  • 🔑
    CRON_SECRET en Vercel El mismo valor que generaste en el paso 4 debe estar en las variables de entorno de Vercel. El endpoint lo valida con process.env.CRON_SECRET.
  • 🧪
    Probar manualmente Puedes disparar el cron desde el dashboard de Vercel → Cron Jobs → Run Now, o con curl usando tu CRON_SECRET. 
# Disparar el cron manualmente para probar
$ curl -X GET https://tu-app.vercel.app/api/cron/sync \
  -H "Authorization: Bearer TU_CRON_SECRET"
ℹ️

Plan Vercel Hobby: los cron jobs están disponibles pero con un límite de 2 crons y ejecución mínima de 1 vez por hora en algunos planes. Si necesitas cada 5 minutos, activa el Cron Jobs add-on o usa el plan Pro.

7

Dominio personalizado

Apuntar tu propio dominio a la app desplegada
  • 1️⃣
    Agregar dominio en Vercel En tu proyecto → Settings → Domains → Add. Escribe tu dominio, por ejemplo wc26pool.com o pool.midominio.com.
  • 2️⃣
    Configurar DNS Vercel te dará un valor CNAME o un registro A. Agrégalo en tu proveedor de DNS (GoDaddy, Namecheap, Cloudflare, etc.). La propagación toma entre 5 minutos y 48 horas.
  • 3️⃣
    SSL automático Vercel provisiona un certificado SSL/TLS vía Let's Encrypt automáticamente. No necesitas configurar nada extra.
  • 4️⃣
    Actualizar Supabase Auth Agrega tu nuevo dominio en Supabase → Authentication → URL Configuration: actualiza Site URL y agrega el dominio a Redirect URLs.
8

Checklist post-deploy

Verifica que todo funciona antes de compartir el link
  • La URL de producción carga sin errores 500 en la consola del servidor (revisa Vercel → Functions logs)
  • El registro de usuario funciona y crea un perfil en Supabase → Table Editor → profiles
  • El login redirige al dashboard correctamente
  • La página de Partidos muestra los 72 partidos de grupos con fechas y banderas
  • Se puede guardar una predicción (aparece en Supabase → predictions_groups)
  • El Bracket queda bloqueado hasta completar los 72 partidos de grupos
  • La página de Quinielas permite crear y unirse a una quiniela con código
  • La Tabla muestra la posición de los usuarios de la quiniela activa
  • El cron /api/cron/sync responde 200 cuando se llama con el CRON_SECRET correcto
  • Supabase → Authentication → URL Configuration tiene el dominio de producción en Redirect URLs
  • SUPABASE_SERVICE_ROLE_KEY NO aparece en el código del cliente (verifica con DevTools → Network)
✅

¡Listo para el Mundial! Si todos los puntos están marcados, la app está lista. Comparte el link de invitación con tus jugadores desde /pools.

Monitoreo en producción

  • 📊
    Logs de Vercel Vercel → tu proyecto → Functions muestra los logs de Server Actions y API routes en tiempo real. Ideal para detectar errores en producción.
  • 🛢️
    Supabase Dashboard En Database → Table Editor puedes ver y editar datos directamente. En Logs → API ves todas las queries en tiempo real.
